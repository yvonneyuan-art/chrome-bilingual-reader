# Chrome Bilingual Reader

一个 Chrome 扩展，用来在 Chrome 将英文网页翻译成中文后，继续显示英文原文，方便中英对照阅读。

Chrome 自带翻译很方便，但翻译完成后原文通常就看不到了。这个扩展会在翻译前保存页面中的英文文本，然后在页面翻译成中文后，把保存的英文原文显示回对应位置。

本扩展主要适用于博客、文档、README、新闻、论文说明页等结构相对稳定的长文网页。不适用于 X/Twitter、信息流、聊天窗口、实时更新看板、无限滚动列表等动态更新网页。

## 功能

- 在 Chrome 翻译前捕获页面中的英文原文。
- 在中文翻译页面中显示英文原文，形成中英对照。
- 尽量保留常见阅读结构，例如标题、段落、列表和表格。
- 跳过独立代码块，但保留包含内联代码的说明句。
- 尽量保留多行文本的换行。
- 按页面 URL 保存捕获结果，刷新或重新访问同一页面后可尝试恢复对照显示。
- 不调用外部翻译 API，配合 Chrome 自带翻译离线工作。

## 安装

目前项目以“加载已解压的扩展程序”的方式使用。

1. 克隆或下载这个仓库。
2. 打开 Chrome，进入 `chrome://extensions`。
3. 打开右上角的 **开发者模式**。
4. 点击 **加载已解压的扩展程序**。
5. 选择仓库中的 `chrome-extension` 文件夹。

如果你更新了扩展文件，需要在 `chrome://extensions` 中点击该扩展的重新加载按钮，然后刷新正在测试的网页。

## 使用

1. 打开一个英文网页。
2. 点击扩展图标。
3. 在翻译前点击 **捕获英文原文**。
4. 使用 Chrome 自带翻译将页面翻译成中文。
5. 再次点击扩展图标。
6. 点击 **显示中英对照**。

如果显示位置不理想，可以刷新页面，点击 **清除本页缓存**，然后重新执行捕获和翻译流程。

## 隐私

这个扩展不会调用任何外部翻译服务。捕获到的页面文本会按页面 URL 保存在你的本地 Chrome 扩展存储中，用于在同一页面恢复中英对照显示。

扩展使用的权限：

- `activeTab`：与当前正在查看的网页通信。
- `storage`：按页面 URL 在本地保存捕获到的文本。
- `<all_urls>`：允许扩展在普通网页上运行。

## 已知限制

- 必须在 Chrome 翻译为中文之前先点击插件的 **捕获英文原文**。
- 图片、视频、Canvas、SVG、跨域 iframe 中的文字无法作为普通页面文本读取。
- 动态网页可能在捕获后改变 DOM 结构，影响对齐效果。
- 不支持 X/Twitter、信息流、聊天窗口、实时更新看板、无限滚动列表等动态更新网页。这类页面会不断新增、替换或回收页面节点，容易导致原文缓存失效、内容错位或显示异常。
- 如果页面长时间没有刷新，或扩展刚刚重新加载，当前页面里的扩展脚本可能会和浏览器扩展断开连接。此时点击按钮可能会提示无法连接当前页面，需要刷新网页后重新捕获英文原文。
- `chrome://` 这类浏览器内部页面不允许普通扩展内容脚本运行。

## 项目结构

```text
chrome-extension/   Chrome 扩展源码
tools/              本地辅助脚本，例如图标生成脚本
skills/             可选的 Codex 维护 skill，用于记录这个项目的维护经验
```

## 开发

这个扩展使用 Manifest V3，代码是普通 HTML、CSS 和 JavaScript，不需要构建步骤。

常用检查命令：

```bash
node --check chrome-extension/content.js
node --check chrome-extension/popup.js
python3 -m json.tool chrome-extension/manifest.json
```
